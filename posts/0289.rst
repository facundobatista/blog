En Python decidimos hace tiempo irnos de `SourceForge <http://sf.net/>`_. El primer paso fue migrar el repositorio de código, desde ahí a `nuestro server <http://svn.python.org/>`__. Y hace algunos días, terminamos migrando el tracker.

O sea, sacamos toda la administración de bugs y parches y los pusimos en `nuestro server <http://bugs.python.org/>`__, manejándolos con `Roundup <http://roundup.sourceforge.net/>`_ (adivinen con qué está hecha este administrador, ;).

Yo tenía una herramienta que iba a SourceForge y hacía un resumen de los tickets (había hablado de eso `acá </posts/0246>`_). Obviamente la tuve que cambiar, y acabo de terminar de hacerlo.

Como ahora los bugs y parches están en la misma lista (son, genéricamente, *tickets*), quedó genericamente `un sólo resumen <http://www.taniquetil.com.ar/facundo/py_tickets.html>`_. Obviamente, dejé este link en la barra derecha...
