Releases por dos: Encuentro y fades
<tags>



Estos últimos días se liberaron nuevas versiones de dos proyectos en los que estoy involucrado activamente.

A principio de mes lancé Encuentro 3.1 (como ya sabrán, este programa permite buscar, descargar y ver contenido del Canal Encuentro, Paka Paka, BACUA, Educ.ar y otros).

La versión 3.1 trae los siguientes cambios con respecto a la versión anterior:
<ul>
<li>Vuelve a funcionar luego de los cambios de backend de Encuentro y Conectate </li>
<li>Ahora con CTRL-F se va directamente al campo de filtro (gracias <a href="https://twitter.com/emilianohf">Emiliano</a>) </li>
<li>Se rehizo el manejo de la lista de episodios: ahora verlos y filtrarlos es muchísimo más rápido</li>
<li>Mejoras en el empaquetado, debería funcionar para muchas (todas?) las versiones de Debian/Ubuntu (gracias <a href="https://twitter.com/alvesadrian">Adrián Alves</a>).  </li>
<li>Varias mejoras al encontrar nuevos episodios de los distintos backends, y correcciones en general. </li>
</ul>

Más info y cómo descargarlo, instalarlo, etc, en la <a href="http://encuentro.taniquetil.com.ar/">página oficial</a>.

Por otro lado, ayer se lanzó fades 3 (un proyecto orientado a desarrolladores Python, en contraposición a Encuentro que está pensado para el usuario final), que desarrollamos principalmente <a href="https://twitter.com/gilgamezh">Nico Demarchi</a> y yo.

fades (en inglés: FAst DEpendencies for Scripts) es un sistema que maneja automáticamente los virtualenvs en los casos simples que uno normalmente encuentra al escribir scripts o programas pequeños.  Crea automáticamente un nuevo virtualenv (o reusa uno creado previamente) instalando las dependencias necesarias, y ejecutando el script dentro de ese virtualenv.

¿Qué hay de nuevo en esta release?
<ul>
<li>Podés usar diferentes versiones del intérprete: simplemente pasá <span style="font-family: courier new,courier;">--python=python2</span> o lo que te convenga.</li>
<li>Las dependencias pueden especificarse en la linea de comando: no hay necesidad de cambiar el script para una prueba rápida, simplemente especificá la dependencia necesaria con <span style="font-family: comic sans ms,sans-serif;">--dependency</span>.</li>
<li>Modo interactivo: es la manera más rápida de probar una nueva biblioteca. Sólo hacé <span style="font-family: courier new,courier;">fades -d <dependencia></span> y te abrirá un intérprete interactivo dentro de un venv con esa dependencia.</li>
<li>Soporta tomar argumentos desde el shellbang. De esta manera podés crear un script y poner al principio del mismo algo como: <span style="font-family: courier new,courier;">#!/usr/bin/env fades -d <dependencia> --python=python2.7</span></li>
<li>Puede parsear requerimientos desde un archivo. No hay necesidad de ningún cambio si ya tenés un archivo requirements.txt: simplemente indicalo con <span style="font-family: courier new,courier;">--requirement</span>.</li>
<li>Si no se especifica el repo, toma PyPI por defecto, lo que resulta en código más limpio y simple.</li>
<li>Tiene una base de datos integrada para conversiones típicas de nombres: de esta manera se puede marcar con fades un "import bs4" incluso si ese no es el nombre del paquete en PyPI.</li>
<li>Otros cambios y correcciones menores.</li>
</ul>

Toda la info, en la <a href="https://pypi.python.org/pypi/fades">página de PyPI del proyecto</a>. |
